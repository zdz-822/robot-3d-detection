# 实验记录规范

每次训练在开始前创建一份实验记录。不要只保存最终最优指标；失败实验和显存溢出配置同样需要记录。

| 字段 | 内容 |
| --- | --- |
| 实验编号 | 例如 `E001` |
| 日期 | 训练开始日期 |
| Git commit | 代码版本 |
| 数据版本 | KITTI 或自采数据版本与划分 |
| 模型 | PointPillars / CenterPoint / 改进版本 |
| 改动 | 相比基线的唯一变化 |
| GPU 与显存 | 设备型号、batch size、峰值显存 |
| 训练耗时 | 总耗时、epoch 数 |
| 结果 | 各类别 AP、Recall、FPS、端到端延迟 |
| 结论 | 是否保留该改动及原因 |

## 结果目录

训练输出放到本地 `outputs/`，目录命名采用：

```text
outputs/<experiment-id>-<model>-<yyyy-mm-dd>/
```

每个目录至少包含训练配置副本、命令行、指标 JSON/文本、日志和代表性可视化图像。
