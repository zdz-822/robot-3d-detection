# E009: 连续回放稳定性与延迟评测

## 目标

将单次 ROS 2 时序检测演示升级为可重复的稳定性评测。评测自动重复运行 G1 传感器级仿真，并在每轮检查是否完整回放、是否在三帧缓存后持续发布检测，以及推理延迟分布。

## 运行方式

在 WSL 项目目录中执行：

```bash
bash scripts/run_e009_stability_benchmark.sh 3
```

第一个参数是重复轮次，默认 `3`。每轮产生 12 帧输入，其中前 2 帧用于时序缓存，后 10 帧应产生检测结果。原始日志、轮次时间表与自动生成报告会写入被 Git 忽略的 `artifacts/e009_stability/<时间戳>/`。

## 指标

- 完整轮次：退出码为 0、日志包含 `Replay complete` 且恰好发布 10 个检测帧。
- 全部延迟：包含每轮首次 CUDA 预热，反映冷启动体验。
- 稳定延迟：排除每轮首次检测，反映连续运行性能；报告 P50、P95 和最大值。
- 平均目标数：用于发现检测输出异常为空或异常激增。

## 实际结果

2026-07-17 在 RTX 4060 Laptop GPU 上完成 3 轮连续回放，每轮使用相同的 12 帧 nuScenes mini 场景：

- `3/3` 轮完整结束；每轮均在前 2 帧缓存后发布 10 帧检测，共 `30` 个检测帧，无缺帧。
- 全部检测延迟 P50 / P95 / 最大值为 `116.3 ms / 642.7 ms / 811.3 ms`。P95 偏高主要来自每轮首次 CUDA 预热。
- 排除每轮首次检测后的稳定延迟 P50 / P95 / 最大值为 `113.7 ms / 136.4 ms / 143.9 ms`。
- 每帧平均检测 `8.2` 个目标，三轮结果一致；单轮墙钟时间为 `31-34 s`，其中包含模型加载与启动等待。

原始日志、CSV 和自动生成报告保存在本机 `artifacts/e009_stability/20260717-203949/`，该目录不提交到仓库。

## 边界

本实验验证固定 nuScenes mini 场景中的软件链路稳定性，不代表真实 G1 雷达、真实网络和真实环境中的最终性能。真实上机后应以真实 rosbag 重复执行同样的流程。
